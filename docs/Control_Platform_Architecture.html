Miert Agentic RAG

A hagyomanyos AI megkozelites: a rendszer elore osszeallitja a kontextust, es odaadja az LLM-nek. A problema: vagy tul sokat gyujt (lassu, draga), vagy tul keveset (rossz valasz).

Az agentic pattern megforditja: az LLM dont, mikor mit kerdez le. A rendszer tool-okat biztosit — az LLM hivja oket igeny szerint, iterativan melyul.

Pelda: „Miert nott az IT koltseg?”
→ get_budget_summary(IT) → latja: CloudHost kiugro
→ get_partner_detail(CloudHost) → 3 szamla 48h alatt, +200%
→ Valasz: „Az IT +12%-a a CloudHost miatt. 3 szamla 48h alatt, 4.2M Ft.”

A rendszer nem tudta elore, hogy a CloudHost partnerre is szukseg lesz. Az LLM dontott.

Biztonsag
  • Rate limiting — nginx szintu (login: 5r/m, agent: 5r/m)
  • SQL injection vedelem — READ ONLY tranzakcio + bovitett tiltolista
  • RBAC — 5 role (admin, cfo, dept_head, accountant, reviewer)
  • JWT auth — Bearer token, 8h lejarat
  • HTTPS — Let's Encrypt SSL, auto-renewal
  • Secrets — GitHub Secrets, .env kizarva git-bol
